# 单一弹唱界面重构设计

> 日期：2026-07-21
> 状态：已确认，直接实施
> 决策记录：页面内嵌上传 + PRD 理想全量功能 + 遗留代码直接删除 + 直接完整实现

## 背景

当前前端是 7 视图 SPA（home / analysis / overview / player / focus / results / library），
由 `index.html` + `src/product-app.js`（1647 行）驱动，另有一套遗留的五面板 demo 栈
（`main.js` / `app.js` / `ui-demo.js` / `features/*`）。用户要求砍掉所有页面，
只保留一个对准弹唱跟练的界面，功能按 PRD 理想全量实现。

## 目标架构

### 页面结构（单 view，无路由）

```
┌────────────────────────────────────────────────────┐
│ 弦间 │ 曲目名 · BPM │ 速度 50–100% │ 🎙状态 │ 设置  │
├──────────────────────────────┬─────────────────────┤
│        原始教学视频           │  左手特写 / 右手特写  │
├──────────────────────────────┴─────────────────────┤
│ 波形 → 小节 → 和弦轨 → 六线谱（点击任意元素互跳）    │
├────────────────────────────────────────────────────┤
│ 实时判定 · 得分 · 错误提示 · 循环 · 重练             │
└────────────────────────────────────────────────────┘
```

- **空状态**：页面中央为上传区（拖拽 / 选择视频 / 示例课程）。
- **解析中**：上传区原地变为 8 步解析进度，完成后原地展开练习布局。
- **纠错模式（focus）**：页面内覆盖层，不跳页。含老师动作回看、逐帧、镜像、
  AI 标记开关、降速阶梯（60→75→90→100%）、轮次目标、"我来试试"。
- **练习结果（results）**：页面内覆盖层。得分环、三项指标、掌握地图、前后对比。

### 功能接线

| 能力 | 实现 |
|---|---|
| 上传 / 解析 | 复用现有后端上传 + 轮询 + `applyScore`，进度 UI 内嵌 |
| 实时判定 | `core/audio/detector.js` + `core/matching/*` 真实接入，替换模拟反馈 |
| 练习引擎 | `core/practice/session.js` + `loop.js` + `speed.js` 驱动纠错闭环 |
| 多模态同步 | `core/video/sync.js` 统一时间轴，谱面/波形/和弦/视频点击互跳 |
| 双手特写 | 保留现有 UI 结构，按可信度降级展示 |
