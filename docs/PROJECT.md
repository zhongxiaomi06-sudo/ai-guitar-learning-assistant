# PROJECT.md — 吉他 AI 跟弹系统

> 产品阶段：MVP  
> 目标平台：Web 桌面端优先，移动端后续  
> 核心场景：用户通过吉他教学视频跟弹，系统实时监听并给出游戏化/KTV 式反馈，帮助用户完整弹出目标片段  
> 版本：v1.0  
> 最后更新：2026-07-21

---

## 1. 产品概述

### 1.1 产品定位

吉他 AI 跟弹系统是一款面向吉他爱好者的“演奏辅助”产品。用户导入一段吉他教学视频（本地上传或 URL 抓取），系统自动或半自动地生成可交互谱面，并将谱面与视频时间精确对齐。用户跟随视频演奏时，系统通过麦克风实时监听，以音游/KTV 式的可视化反馈和简洁文字建议，帮助用户把目标片段“弹出来”。

### 1.2 核心价值

- 把普通吉他教学视频变成可定位、可跟弹、可反馈的互动课程。
- 不追求乐理理解，只关注声音结果：音高、节奏、和弦是否弹对。
- 发现错误后，自动降速、循环难点、跳回视频对应位置，快速闭环纠正。
- 基于谱面自动生成手型示意图，告诉用户左右手该做什么动作。

### 1.3 MVP 产品原则

- Web 桌面端先行，验证完整演奏闭环后再做移动端。
- 自动扒谱与真实手部识别暂时搁置，MVP 使用人工导入或预精修谱面数据。
- 手部动作不通过 AI 识别，而是基于谱面生成示意图。
- 实时判定采用教学友好型容错，避免普通麦克风环境下频繁误报。
- 路演 DEMO 视频使用预精修谱面与时间轴，但前端不暴露“预制数据”概念。
- 普通视频支持真实解析流程，低可信度结果降级展示。

## 2. 产品目标与非目标

### 2.1 MVP 目标

1. 用户可以本地上传或粘贴 URL 导入吉他教学视频。
2. 系统能够加载并展示谱面（人工导入/预精修）。
3. 每个音符、和弦和小节都具有对应的视频时间点。
4. 视频、声音、谱面能够同步播放与定位。
5. 基于谱面生成左手按弦与右手拨弦示意图。
6. 用户可通过麦克风进行实时跟弹。
7. 系统可识别明显错音、错和弦、漏音和节奏偏差。
8. 错误能够定位到具体音符、拍位和小节。
9. 系统可自动生成降速、拆句、循环和提速练习。
10. 用户能够完成“跟弹—纠错—重练—达标—恢复原速”的完整闭环。

### 2.2 MVP 非目标

- 全自动 AI 扒谱（技术调研中，暂不实现）。
- AI 识别真实左右手动作与手指落点。
- 乐理教学、导师工作台、课程收费体系。
- 社区、评论、排行榜、多人合奏。
- 人声与伴奏高精度分离。
- 电吉他效果器与音色识别。
- 三机位独立视频上传。
- 复杂五线谱编辑器。
- 电视端、多语言和社交分享体系。
- 云端账号同步与课程商城。

## 3. 目标用户与使用环境

### 3.1 目标用户

- 正在通过视频学习吉他的初级或中级用户。
- 能够模仿老师动作，但难以判断自己具体错在哪里的用户。
- 需要反复练习和弦转换、单音旋律、拨弦或节奏片段的用户。
- 希望像“音游”一样快速把一首歌弹出来的用户。

### 3.2 典型使用环境

- 用户在电脑前使用桌面端 Web。
- 电脑播放教学视频并显示谱面与手型图。
- 用户使用木吉他或电吉他演奏。
- 浏览器通过电脑麦克风或外接麦克风监听声音。
- 使用环境可能存在轻微噪声、琴弦共振和设备输入延迟。

## 4. 支持范围与输入约束

### 4.1 视频格式

- 支持 MP4、MOV。
- 建议时长为 30 秒至 10 分钟。
- 建议文件大小不超过 1 GB。
- 视频应包含可辨识的吉他声音。
- 单人、单把吉他、左右手基本可见时效果最佳。

### 4.2 输入质量检查

上传或导入后自动检查：

- 是否包含音轨。
- 音量是否足够。
- 环境噪声是否过高。
- 视频是否严重模糊、过暗或抖动。
- 视频时长/大小是否超限。

质量不足时允许用户继续，但必须明确说明哪些功能可能降级。

### 4.3 导入方式

- 拖拽本地文件上传。
- 点击选择文件。
- 粘贴 URL，后端调用 yt-dlp / you-get 等框架抓取。
- 系统内置预精修 DEMO 视频，通过内容指纹匹配直接加载。

## 7. 功能需求与呈现形式

### 7.1 上传/导入页

#### 页面目标

让用户立即理解：导入一段吉他视频，即可生成带谱面、手型图和实时纠错的互动跟弹课程。

#### 页面布局

- 顶部：产品 Logo、名称、麦克风状态。
- 主标题：“导入吉他视频，开始 AI 跟弹”。
- 副标题：“支持本地上传、视频链接或选择 DEMO 视频”。
- 大型拖拽上传区域。
- URL 输入框与“抓取”按钮。
- DEMO 视频选择区（封面 + 标题）。
- 支持格式、时长、文件大小说明。

#### 选择文件后的状态

上传区域切换为视频信息卡：

- 视频封面。
- 文件名/URL。
- 时长、文件大小。
- 音频质量检测。
- 视频清晰度检测。
- “开始解析”按钮。
- “重新选择”按钮。

#### 异常反馈

- 无音轨：阻止解析，提示更换含声音的视频。
- 音量过低：允许继续，但提示识别准确度可能降低。
- 视频过长或过大：提示压缩或截取片段。
- 格式不支持：明确支持格式并提供转换建议。
- URL 抓取失败：提示检查链接或本地上传。

### 7.2 解析进度页

#### 页面目标

用逐步生成的内容呈现 AI 工作过程，避免用户面对只有百分比的等待页面。

#### 页面布局

- 左侧：播放上传视频。
- 右侧：纵向分析步骤与当前进度。
- 底部：逐步生成的波形、小节线、和弦和谱面预览。

#### 解析步骤

1. 提取视频与音频。
2. 识别吉他声音。
3. 检测速度、节拍与小节。
4. 加载谱面数据。
5. 对齐谱面与音频。
6. 生成手型图。
7. 拆分练习片段。

每步显示图标、状态、进度和一条简短结果：

```text
✓ 音频提取完成
✓ 检测到速度：92 BPM
✓ 检测到拍号：4/4
● 正在加载谱面
○ 等待生成手型图
```

#### 解析完成摘要

- 小节数量。
- 音符数量。
- 已识别和弦。
- BPM 与拍号。
- 综合解析可信度。
- “开始练习”主按钮。

### 7.3 核心跟练页

#### 页面布局

```text
┌─────────────────────────────────────────────────────────────┐
│  曲目 / 当前片段 / BPM / 速度 / 麦克风 / 设置 / 开始暂停   │
├──────────────────────┬──────────────────────────────────────┤
│                      │                                      │
│   视频面板            │           乐谱面板                    │
│                      │           （六线谱 + 手型图）          │
├──────────────────────┴──────────────────────────────────────┤
│  匹配面板（音游下落 / KTV 音高条）                          │
├─────────────────────────────────────────────────────────────┤
│  建议面板  │  设置面板                                      │
│  反馈/统计  │  速度/模式/难度/设备/主题                      │
└─────────────────────────────────────────────────────────────┘
```

#### 视频面板

- 播放原视频。
- 显示当前播放进度、小节/拍子。
- 支持拖动进度条跳转。
- 支持 0.25x–1.0x 连续调速，保持音高。
- 支持 A/B 片段循环。
- 支持全屏。
- 当用户弹错时，自动回跳至该小节开始。

#### 乐谱面板

- 六线谱形式，支持滚动。
- 当前播放位置高亮。
- 已完成音符降低透明度。
- 后续音符保持中性色。
- 播放指针与视频同步移动。
- 点击任意音符：
  - 视频跳到该音符前约 0.8 秒。
  - 显示对应手型图。
  - 允许从该音符创建循环。

每个音符事件包含：

- 开始和结束时间。
- 琴弦与品位。
- 音高。
- 所属和弦。
- 拍位与小节。
- 对应视频时间。
- 对应左手和右手动作。

#### 手型图区域

- 左手按弦图：指板品位、手指编号、横按范围。
- 右手拨弦图：拨弦方向、扫弦范围、使用手指（P/i/m/a）。
- 手型图基于谱面自动生成，不识别真实视频中的手型。

#### 和弦轨

- 显示在六线谱上方或独立区域。
- 当前和弦展开：和弦名称、标准和弦图。
- 当视频使用转位、省略音或替代指法时，优先展示谱面定义指法。

#### 匹配面板

**音游模式**：

- 6 条轨道对应吉他 6 根弦。
- 音符从上方下落，到达判定线时用户弹奏。
- 判定：Perfect / Good / Miss。
- 显示连击数、准确率。

**KTV 模式**：

- 目标音高曲线（蓝色）与用户实时音高曲线（绿色/红色）。
- 偏离时曲线分离，命中时重合。
- 适合单音旋律练习。

用户可在设置中切换，系统根据谱面自动推荐默认模式。

#### 建议面板

- 当前命中状态：Perfect / Good / Miss。
- 实时文字提示：
  - “完美！”
  - “第 3 弦偏高，检查品位”
  - “漏了第 5 弦空弦音”
  - “快了 120ms，稳住节奏”
  - “准备下一小节：Am 和弦”
- 统计：准确率、连击数、本次片段完成度。
- 连续错误自动暂停并提示重试。

#### 设置面板

- 播放速度：0.25x–1.0x，步进 0.05x。
- 匹配模式：音游 / KTV。
- 难度：宽松 / 标准 / 严格（判定容差）。
- 自动调速开关。
- 麦克风设备选择。
- 主题：浅色/深色。
- A/B 循环设置：手动输入或自动围绕错误小节。

### 7.4 谱面—声音—动作同步

系统为每个演奏事件建立统一时间对象：

```text
事件编号：note_024
所属小节：第 6 小节
所属拍位：第 2 拍
视频时间：00:18.420
音频时间：00:18.398
音高：E4
琴弦：1 弦
品位：空弦
所属和弦：C
左手动作：按 C 和弦（图示）
右手动作：食指上拨（图示）
```

播放到该事件时：

- 六线谱音符高亮。
- 和弦轨切换。
- 原视频到达对应动作。
- 左手手型图更新。
- 右手拨弦图更新。
- 匹配 UI 显示目标音符。

点击视频进度条、谱面音符、和弦或错误记录中的任意一项，其余视图同步定位。

### 7.5 跟练准备

#### 麦克风授权

首次跟练时显示轻量授权面板：

```text
需要使用麦克风监听你的吉他演奏
录音仅用于实时分析
```

操作：

- “允许麦克风”。
- “暂不开启”。

#### 环境检查

授权后监听约 3 秒，检测：

- 环境噪声。
- 输入音量。
- 是否存在吉他声音。
- 输入延迟。

结果示例：

```text
麦克风已连接
吉他音量正常
环境噪声较低
预计监听延迟：42 ms
```

#### 快速校准

系统显示四拍倒计时，并要求用户弹一次指定空弦，用于估算：

- 输入延迟。
- 音量阈值。
- 环境共振。

完成后显示“3、2、1、开始”。

### 7.6 实时监听与教学友好型反馈

#### 判定原则

- 以帮助练习为目标，不追求录音棚级判定。
- 对轻微琴弦共振、背景噪声和设备延迟设置容错。
- 对明显错误及时反馈，对不确定结果降低结论强度。
- 连续演奏时不使用频繁弹窗打断用户。

#### 正确音符

- 音符短暂变绿。
- 显示轻微确认动画。
- 不弹出额外面板。

#### 错音

当前音符变红，并显示目标与实际结果：

```text
目标：2 弦 1 品 C
听到：2 弦 3 品 D
可能原因：手指按高了一品
```

#### 错和弦

当前和弦框变红，并展开差异：

```text
目标：C
检测结果：Am7
缺少：1 弦空弦 E
多出：3 弦空弦 G
```

#### 漏音

目标音符显示为空心橙色：

```text
1 弦没有清晰发声
可能被左手误触消音
```

#### 节奏偏差

- 提前：音符左侧显示方向箭头。
- 延后：音符右侧显示方向箭头。
- 准时：显示中心圆点。
- 明显偏差可显示毫秒数，例如“慢了 120 ms”。

轻微偏差不触发错误。容错范围随当前速度和片段难度调整。

#### 连续播放反馈

练习过程中只使用：

- 颜色。
- 小图标。
- 一行短提示。
- 小节状态。
- 实时得分。

详细错误分析在乐句结束或连续发生相同错误时出现。

### 7.7 错误定位与练习总结

乐句结束后，谱面生成错误热力图：

- 绿色：正确。
- 黄色：轻微节奏偏差。
- 橙色：漏音或发音不清晰。
- 红色：错音或错和弦。
- 灰色：未检测到有效演奏。

总结卡示例：

```text
本次练习：第 5–8 小节

音符正确率：86%
节奏稳定度：78%
和弦完整度：82%

主要问题：
1. 第 6 小节第 2 拍：换到 C 和弦时漏掉 1 弦
2. 第 7 小节第 4 拍：进入早了 160 ms
3. 第 8 小节：G → Em 换把速度不足
```

点击问题后：

- 谱面定位到具体错误音符。
- 视频跳到老师对应时间点。
- 手型图同步切换。
- 自动创建错误前后的短循环。

### 7.8 自动难点纠错

#### 触发条件

- 同一目标连续出现两次相同错误。
- 单次出现严重错和弦或整拍漏弹。
- 系统识别到用户在某次换把处明显停顿。
- 用户主动点击错误记录或谱面音符。

提示示例：

```text
第 6 小节的 C 和弦连续出现 2 次漏音
建议进行 20 秒专项练习
```

操作：

- “开始专项练习”。
- “再完整弹一次”。
- “暂时忽略”。

#### 自动拆句

一般错误默认截取：

- 错误前 1 小节。
- 错误所在小节。
- 错误后 1 小节。

单次换把问题可缩短为动作前后约 2–4 秒。

#### 自动降速

默认速度阶段：

```text
60% → 75% → 90% → 100%
```

若用户在 60% 速度仍连续失败，可降至 50%。降速时保持原始音高。

#### 循环结构

每一轮包含：

1. 观看老师演示。
2. 用户模仿演奏。
3. AI 快速判断。
4. 进入下一轮或重新观看。

状态示例：

```text
第 2 / 4 轮
当前速度：60%
目标：连续正确 2 次
```

### 7.9 老师动作回看

进入纠错模式时，视频自动跳到对应动作。播放范围应包含：

- 错误音符前约 1 秒的动作准备。
- 错误动作本身。
- 动作后约 0.5 秒的衔接。

#### 左手动作标记

在手型图中显示：

- 手指落点。
- 保留不动的手指。
- 换把起点与终点。
- 手腕移动方向。

示例提示：

```text
不要完全抬起食指
保留食指位置，只移动中指和无名指
```

#### 右手动作标记

在手型图中显示：

- 目标琴弦。
- 使用手指（P/i/m/a）。
- 上拨或下拨。
- 扫弦起点与终点。
- 应避开的琴弦。

#### 专用控制

- 逐帧查看。
- 只听老师演奏。
- 只听用户上一次演奏。
- 老师与用户录音切换比较。

### 7.10 再次监听与前后比较

用户点击“我来试试”后，系统重新倒数并监听相同片段。

完成后显示前后对比：

```text
                  第一次     本次
音符正确率         72%  →   91%
节奏稳定度         68%  →   84%
和弦完整度         60%  →   94%
```

错误事件对比示例：

```text
第 6 小节第 2 拍
上次：1 弦漏音
本次：正确
```

系统根据结果给出单一、明确的下一步：

- 错误已解决：进入提速。
- 偶发错误：当前速度再练一次。
- 相同错误：重新观看老师动作。
- 节奏仍不稳：打开节拍器并缩短片段。
- 多种错误同时出现：降低速度。

### 7.11 达标与恢复原速

#### 单轮达标标准

- 关键音符正确率不低于 90%。
- 和弦完整度不低于 85%。
- 明显节奏偏差不超过 2 次。
- 目标错误连续正确 2 次。

#### 提速状态

```text
60% 速度：通过
75% 速度：通过
90% 速度：练习中
100% 速度：待解锁
```

用户可以：

- 降低速度。
- 保持当前速度。
- 跳过本次提速。
- 返回原速尝试。

#### 连续失败处理

连续两次未达标：

- 保持当前速度。
- 缩短循环片段。
- 只突出最重要的一个错误。

连续四次未达标：

- 再降低一级速度。
- 重新播放老师动作。
- 显示手型图提示。
- 建议进行无节奏慢动作练习。

### 7.12 完整片段完成页

所有难点通过后，邀请用户以原速完成整个片段。

结果页展示：

- 总体得分。
- 音符正确率。
- 和弦完整度。
- 节奏稳定度。
- 已掌握小节。
- 仍需练习的小节。
- 本次练习时长。
- 相比首次练习的提升。

掌握地图示例：

```text
第 1–4 小节：已掌握
第 5–8 小节：已掌握
第 9–12 小节：基本掌握
第 13–16 小节：已掌握
```

操作：

- “再弹一次”。
- “继续下一片段”。
- “复习薄弱小节”。
- “返回课程”。

### 7.13 我的课程

课程卡显示：

- 视频封面。
- 曲目名称。
- 总时长。
- 学习进度。
- 已掌握片段。
- 最近练习时间。
- “继续练习”按钮。

课程状态包括：

- 正在解析。
- 解析失败。
- 可以开始。
- 练习中。
- 已完成。

系统保存：

- 上次播放位置。
- 用户速度设置。
- 自定义循环片段。
- 错误历史。
- 薄弱小节。
- 最佳成绩。

### 9.1 视频处理模块

负责文件上传、URL 抓取、格式验证、转码、音轨提取、封面生成、片段切割和降速播放资源生成。

### 9.2 音乐分析模块

负责 BPM、拍号、小节、起音、音高、和弦识别，以及谱面加载与对齐。

### 9.3 手型生成模块

基于谱面音符与和弦，生成左手按弦与右手拨弦示意图，不依赖视频动作识别。

### 9.4 多模态时间轴模块

将原视频、音频波形、音符、和弦、小节、手型图映射到统一时间轴，并提供双向定位能力。

### 9.5 实时演奏检测模块

负责浏览器麦克风采集、环境校准、起音检测、音高与和弦估计、目标谱面对齐和教学友好型判定。

### 9.6 自适应练习引擎

负责错误聚合、难点识别、自动拆句、循环范围、速度调整、达标判断、前后比较和进度保存。

## 10. 状态与异常处理

### 10.1 AI 结果可信度

系统不得在低可信度情况下展示伪精确结论：

- 谱面加载失败：提示使用本地预精修数据或人工导入。
- 对齐可信度低：提示手动调整时间偏移。
- 音频质量差：扩大容错并提示改善环境。

### 10.2 麦克风环境降级

- 环境噪声高：扩大容错并提示改善环境。
- 输入过小：提示用户靠近麦克风。
- 延迟过高：提示关闭蓝牙音频设备或使用有线设备。
- 无法识别吉他：允许重新校准或进入仅观看模式。

### 10.3 上传与解析异常

- 上传中断：支持重新上传。
- 网络中断：提示检查 URL 或本地上传。
- 解析部分失败：保留已完成结果，并说明降级能力。
- 解析完全失败：保留原视频，允许重新解析。
- 页面刷新：能够恢复任务状态和进度。

## 11. 路演方案

### 11.1 演示视频要求

建议使用 45–90 秒的精修视频，包含：

- 清晰左手指板画面。
- 清晰右手拨弦画面。
- 2–4 个常用和弦。
- 一次明显换把。
- 一段单音旋律。
- 一个适合故意制造的漏音点。
- 一个适合故意制造的节奏错误点。

### 11.2 演示流程

1. 在首页上传演示视频或选择 DEMO。
2. 展示输入质量检查。
3. 展示 AI 解析过程（加载谱面、对齐、生成手型图）。
4. 进入自动生成的课程。
5. 点击不同音符，展示视频、谱面与手型图同步跳转。
6. 开启麦克风并完成快速校准。
7. 正确演奏几个音符。
8. 故意弹错一个音。
9. 故意漏掉一个和弦音。
10. 展示系统定位具体音符与小节。
11. 自动跳回老师对应动作。
12. 展示手型图提示。
13. 以 60% 速度循环难点。
14. 第二次正确演奏。
15. 展示练习前后对比。
16. 自动提升到 75% 或 90%。
17. 展示片段掌握结果。

### 11.3 路演稳定机制

- 使用内容指纹匹配演示视频的预精修分析数据。
- 预先保存谱面、时间轴和手型图。
- 解析页按真实资源加载情况推进视觉步骤。
- 麦克风监听和练习反馈保持真实运行。
- 提供稳定演示配置，适当扩大现场噪声容错。
- 演示课程资源支持本地缓存。
- 路演前提供麦克风快速检测。

## 13. 验收标准

### 13.1 上传与解析

- 支持上传 MP4/MOV。
- 支持 URL 自动抓取（合规前提下）。
- 能展示输入质量检查。
- 能展示完整解析过程和中间结果。
- 能加载谱面并生成小节、和弦和六线谱。
- 每个谱面事件都能跳转到对应视频时间。

### 13.2 多模态同步

- 视频、音频波形与谱面播放指针保持同步。
- 点击音符能够定位视频和手型图。
- 手型图跟随谱面事件更新。
- 关键动作能够显示标记。

### 13.3 实时跟练

- 能获取麦克风权限并完成快速校准。
- 能检测明显错音、错和弦、漏音和节奏偏差。
- 能把错误定位到音符、拍位和小节。
- 轻微噪声和琴弦共振不会频繁触发误报。

### 13.4 智能纠错

- 能自动创建错误附近的循环片段。
- 能自动降低播放速度且保持音高。
- 能跳回老师对应动作。
- 能比较两次演奏结果。
- 达标后能逐步恢复原速。

### 13.5 路演质量

- 演示视频的谱面和时间轴分析结果稳定。
- 核心页面无空白状态或占位文案。
- 上传至课程生成期间有连续且可信的视觉反馈。
- 可在 5–8 分钟内完成一次完整闭环展示。

## 14. 成功指标

### 14.1 产品指标

- 上传完成率。
- 解析完成率。
- 从课程生成到开始练习的转化率。
- 完成至少一次纠错循环的用户比例。
- 完成整个练习片段的用户比例。
- 用户重复练习同一课程的次数。

### 14.2 体验指标

- 用户能够理解错误发生在哪个音符和小节。
- 用户能够从手型图回看中获得可执行的纠正方式。
- 用户第二次演奏的目标错误改善率高于第一次。
- 用户不会因为频繁误报而中断练习。

### 14.3 技术指标

- 演示视频时间轴定位误差应控制在肉眼难以感知的范围内。
- 普通视频解析失败时必须给出明确原因和可恢复路径。
- 实时反馈延迟应满足跟练体验，不得明显落后于当前演奏。
- 页面刷新和短暂断网不得丢失解析任务与练习进度。
